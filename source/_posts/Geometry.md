这篇笔记先补充环境贴图和凹凸贴图，再进入几何表示、曲线曲面、网格细分与简化，最后介绍光栅化管线中常用的阴影映射。

这些内容看似分散，其实都围绕一个问题：**怎样用合适的数据结构描述物体，并在有限计算量下获得足够好的形状与视觉细节？**

## 一、纹理的两个补充应用

### 1.1 Environment Mapping

环境贴图用一张或多张二维纹理记录物体周围的光照环境。渲染反射表面时，可以根据反射方向查询环境贴图，近似得到“这个方向上能看见什么”。

常见的表示方式有两种：

#### Spherical Environment Map

把整个环境参数化到一张二维图像上，查询方便，但从球面展开到平面时无法避免形变，靠近极点的位置通常畸变更明显。

#### Cube Map

用立方体的六个面分别记录前、后、左、右、上、下六个方向。查询时根据方向向量选择对应面，再计算面内坐标。它比球面展开更均匀，因此在实时渲染中十分常见。

环境贴图只能描述从某个参考位置观察到的远处环境。它并不会自动计算场景中物体之间真实的多次反射，因此仍是一种近似。

### 1.2 Bump Mapping

凹凸贴图不真正改变模型顶点的位置，而是通过高度纹理的局部变化扰动法线。光照模型接收到不同的法线后，表面看起来就产生了细小起伏。

#### 从二维曲线理解法线扰动

![二维高度变化与法线扰动](/img/posts/geometry/bump-mapping-2d.png)

图中蓝色曲线可以看作一维高度函数 `h(p)`。在位置 `p` 附近，用相邻高度之差估计斜率：

```text
dp = c · [h(p + 1) - h(p)]
```

若原始法线是 `(0, 1)`，曲线的切向量可以近似写为 `(1, dp)`，与它垂直的扰动法线就是：

```text
n' = normalize(-dp, 1)
```

图中的蓝色箭头表示局部切线方向，红色箭头则是与切线垂直的新法线。高度变化越快，法线偏转越明显。

#### 扩展到二维纹理

![二维高度纹理中的法线扰动](/img/posts/geometry/bump-mapping-3d.png)

纹理同时沿 `u` 和 `v` 两个方向变化，因此需要估计两个偏导数：

```text
du = c₁ · [h(u + 1, v) - h(u, v)]
dv = c₂ · [h(u, v + 1) - h(u, v)]

n_local = normalize(-du, -dv, 1)
```

这里得到的是**切线空间中的法线**，还不能直接拿去和世界空间或观察空间的光照方向做点积。实际使用时需要通过 TBN 矩阵，把切线空间法线变换到光照计算所在的坐标空间。

凹凸贴图只改变明暗，不改变轮廓、遮挡关系和真实深度。若希望几何位置也发生变化，则需要位移贴图或曲面细分等方法。

## 二、几何的隐式与显式表示

### 2.1 隐式表示

隐式几何通过一个函数描述表面。例如：

```text
f(x, y, z) = 0
```

所有满足等式的点构成表面。如果 `f` 具有一致的符号约定，还可以用正负判断一个点位于物体内部还是外部。

隐式表示适合做内外测试、布尔运算和碰撞查询，但通常不能直接枚举表面上的全部点，渲染前往往还需要提取网格。

#### Constructive Solid Geometry

![CSG 操作树](/img/posts/geometry/csg-tree.png)

CSG 从球、立方体、圆柱等简单几何体出发，通过集合运算构造复杂形状：

- `A ∪ B`：并集，保留属于任一物体的区域；
- `A ∩ B`：交集，只保留同时属于两个物体的区域；
- `A \ B`：差集，从 A 中挖去 B。

图中从底部基本几何体向上组合，最终得到顶部模型。树形结构记录了“先做什么运算、再和哪个物体组合”，因此也方便修改建模步骤。

#### Signed Distance Function

有符号距离函数记录空间点到表面的最近距离，并用符号区分内部和外部。通常约定表面为 0、外部为正、内部为负。

两个 SDF 的硬布尔组合通常使用：

```text
union:       min(d₁, d₂)
intersection:max(d₁, d₂)
difference:  max(d₁, -d₂)
```

若希望两个物体平滑融合，可以使用 smooth-min 一类函数。直接把两个距离值相加，一般不能得到正确的并集表面。

### 2.2 显式表示

显式几何直接给出表面上的点，或提供从参数域到空间的映射，例如：

```text
(u, v) → (x, y, z)
```

它便于采样、绘制和编辑表面，但判断任意空间点是否位于物体内部通常没有隐式表示直接。

常见形式包括：

- **点云**：保存大量离散采样点，结构简单，但点太稀疏时难以恢复连续表面；
- **多边形网格**：用顶点、边和面描述物体，是实时渲染最常用的表示；
- **参数曲线和曲面**：通过控制点和参数生成连续形状，例如 Bézier 曲线。

## 三、Bézier 曲线

### 3.1 de Casteljau 算法

![de Casteljau 构造二次 Bézier 曲线](/img/posts/geometry/de-casteljau.png)

给定控制点 `b₀、b₁、b₂` 和参数 `t ∈ [0, 1]`，先在相邻控制点之间做线性插值：

```text
b₀¹(t) = (1 - t)b₀ + tb₁
b₁¹(t) = (1 - t)b₁ + tb₂
```

再对新得到的两个点继续插值：

```text
b₀²(t) = (1 - t)b₀¹(t) + tb₁¹(t)
```

最终只剩下的点就是曲线在参数 `t` 处的位置。让 `t` 从 0 变化到 1，就能得到完整曲线。

图中黑色折线是控制多边形，蓝色曲线才是 Bézier 曲线。中间的小点表示 de Casteljau 递归插值得到的过程点。除了首尾控制点外，曲线通常不会经过其他控制点。

### 3.2 Bézier 曲线的性质

![Bézier 曲线的重要性质](/img/posts/geometry/bezier-properties.png)

以三次 Bézier 曲线 `b₀、b₁、b₂、b₃` 为例：

1. **经过首尾端点**：`b(0)=b₀`，`b(1)=b₃`；
2. **端点切线由相邻控制点决定**：`b'(0)=3(b₁-b₀)`，`b'(1)=3(b₃-b₂)`；
3. **仿射不变性**：先变换控制点再生成曲线，与先生成曲线再做同一仿射变换等价；
4. **凸包性质**：整条曲线位于控制点的凸包内。

第三条只保证平移、旋转、缩放、错切等仿射变换。普通 Bézier 曲线不具备一般透视投影下的同样性质；若需要更一般的投影性质，可以使用带权重的有理 Bézier 曲线。

### 3.3 分段 Bézier 曲线与连续性

当控制点较多时，通常使用多段三次 Bézier 曲线拼接，而不是一次构造一个很高次数的多项式。

![分段 Bézier 曲线的控制柄](/img/posts/geometry/piecewise-bezier-continuity.png)

图中的黑点是每一段的端点，蓝色方块与灰色线段表示控制柄。控制柄的位置决定端点处的切线方向和强度。

- **C⁰ 连续**：前一段终点与后一段起点位置相同，曲线不会断开；
- **C¹ 连续**：在 C⁰ 基础上，两侧一阶导数也相同，连接处速度和方向都连续；
- **G¹ 连续**：两侧切线方向共线，但长度可以不同，视觉上平滑，参数速度未必连续。

对于相邻两段三次曲线，若连接点两侧的控制柄共线且长度相等，就能满足常见的 C¹ 拼接条件。

## 四、Bézier 曲面

![张量积 Bézier 曲面的构造](/img/posts/geometry/bezier-surface.png)

Bézier 曲面可以理解为在两个参数方向上分别应用 Bézier 曲线。对于 `4×4` 控制点网格：

1. 固定 `v`，沿 `u` 方向分别计算四条三次 Bézier 曲线；
2. 在这四条曲线上取相同参数 `u` 对应的四个点；
3. 把这四个点作为新的控制点，再沿 `v` 方向计算一次 Bézier 曲线；
4. 改变 `(u, v)`，即可得到整个曲面。

图中灰色曲线表示一个方向上的曲线族，蓝色曲线表示固定某个参数后得到的“移动曲线”，黑点则是当前 `(u, v)` 对应的曲面点。

这里使用的是张量积思想：对两个方向应用 de Casteljau 算法的先后顺序不会改变最终结果。

## 五、曲面细分

细分曲面从较粗的控制网格开始，反复增加顶点并更新位置，使网格逐渐趋向平滑极限曲面。

### 5.1 Loop Subdivision

Loop 细分主要用于三角形网格。每轮操作包含两步：先在边上生成新顶点，再更新原有顶点。

#### 新顶点规则

![Loop 细分中新顶点的位置](/img/posts/geometry/loop-new-vertex.png)

对于内部边 `AB`，设相邻两个三角形的对顶点为 `C、D`，新顶点位置为：

```text
v_new = 3/8(A + B) + 1/8(C + D)
```

图中 A、B 是当前边的两个端点，权重较大；C、D 提供相邻面的形状信息，权重较小。这样比单纯取边中点更能保持曲面趋势。

边界边没有两个对顶点，需要使用单独的边界规则，不能直接套用内部边公式。

#### 旧顶点规则

![Loop 细分中旧顶点的位置](/img/posts/geometry/loop-old-vertex.png)

设旧顶点度数为 `n`，也就是它连接的邻居数量。更新规则可以写为：

```text
v' = (1 - n·u)v + u·Σv_neighbor
```

其中：

```text
u = 3/16,       n = 3
u = 3/(8n),     n > 3
```

图中中心白点是旧顶点，周围六个黑点是度数为 6 时的邻居。更新后的位置是原位置和所有邻居位置的加权平均。

### 5.2 Catmull–Clark Subdivision

Catmull–Clark 可以处理一般多边形网格，并在一次细分后把面逐渐转化为四边形结构。

![Catmull–Clark 的面点、边点和顶点更新](/img/posts/geometry/catmull-clark-rules.png)

一轮细分会计算三类点：

- **面点**：一个面的所有顶点平均；
- **边点**：内部边两端点与相邻两个面点的平均；
- **顶点点**：综合原顶点、相邻边中点和相邻面点得到。

对度数为 `n` 的一般顶点，可写成：

```text
P' = (F + 2R + (n - 3)P) / n
```

其中 `F` 是相邻面点平均，`R` 是相邻边中点平均，`P` 是原顶点。图中的展开公式对应规则网格中度数为 4 的特殊情况。

## 六、网格简化

细分是在增加几何复杂度，而网格简化做相反的事：用更少的三角形近似原始形状。

### 6.1 Edge Collapse

![边坍缩操作](/img/posts/geometry/edge-collapse.png)

边坍缩把一条边的两个端点合并成一个新顶点，并删除因此退化的三角形。图中左侧高亮的边被合并后，右侧多个三角形共享同一个新顶点。

并不是所有边都能安全坍缩。实际算法还需要检查：

- 是否破坏网格的流形结构；
- 是否导致三角形翻转；
- 是否严重破坏边界、尖锐特征或纹理接缝。

### 6.2 Quadric Error Metrics

![二次误差度量](/img/posts/geometry/quadric-error-metrics.png)

如果每次都把新顶点简单放在两个端点的平均位置，误差可能快速积累。二次误差度量为每个候选位置计算它到相关三角形平面的距离平方和：

```text
E(v) = Σ(plane_i(v))²
```

图中左侧的局部平均虽然直观，却可能偏离原始面；右侧方法寻找使总平方距离更小的位置，因此更能保持原几何轮廓。

简化时通常为每条边计算坍缩代价，用优先队列反复选择当前误差最小的边。每次坍缩后，再更新受影响邻域的代价。

## 七、阴影映射

Shadow Mapping 把“某点是否能被光源看见”转换成一次深度比较。

![从光源和相机观察场景的阴影映射](/img/posts/geometry/shadow-mapping.png)

图中顶部是光源，左侧是相机。蓝线表示光源视角下可见的最近表面，红线表示正在判断的光线路径。若相机看到的点在光源深度图中不是最近点，它就被其他几何遮挡，应处于阴影中。

算法分为两遍：

1. 把光源当作相机，从光源视角渲染深度图；
2. 正常从相机渲染，把当前片元变换到光源裁剪空间，查询深度图并比较深度。

判断时不能要求两个浮点深度完全相等，通常使用一个小偏移量：

```text
in_shadow = current_depth > stored_depth + bias
```

`bias` 太小会出现自阴影条纹（shadow acne），太大又会让阴影与物体分离（Peter Panning）。此外，阴影贴图分辨率有限，会产生锯齿；常见改进包括 PCF、多级阴影贴图和更合适的深度偏移策略。

## 八、总结

这一部分可以按照“表示—生成—修改—显示”的顺序串起来：

- 隐式函数、SDF 和 CSG 负责描述形状；
- 网格与 Bézier 曲线曲面显式给出可采样的几何；
- Loop 和 Catmull–Clark 增加网格细节；
- 边坍缩与二次误差度量减少网格复杂度；
- 凹凸贴图用法线变化增加表面细节；
- 阴影映射通过光源视角的深度图解决直接光遮挡。

理解这些方法的关键，不是孤立记忆公式，而是先判断当前问题需要改变的是**真实几何、表面法线、网格密度，还是光照可见性**。不同方法修改的数据不同，能解决的问题也不同。
